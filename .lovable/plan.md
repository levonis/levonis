

## ملخص التغيير

تحديث قسم التسعير ليسمح باختيار أكثر من نوع بيع (حجز مسبق + بيع مباشر) وأكثر من نوع شحن (بحري + جوي) معاً، مع إضافة حقول سعر البيع المباشر والمخزون للخيارات/الألوان عند تفعيل البيع المباشر، ونقل حقول السعر والتكلفة والسعر الأصلي إلى قسم التسعير بالدولار.

---

## التفاصيل التقنية

### 1. تعديل `AdminProductPricingSection.tsx`

**التغييرات:**
- نوع البيع يصبح **checkboxes** بدل radio (يمكن تفعيل حجز مسبق + بيع مباشر معاً)
- نوع الشحن يصبح **checkboxes** بدل radio (يمكن تفعيل بحري + جوي معاً)
- نقل حقول **سعر التكلفة** و**السعر** و**السعر الأصلي قبل التخفيض** إلى داخل هذا القسم
- عند تفعيل الشحن البحري: تظهر حقول CBM
- عند تفعيل الشحن الجوي: تظهر حقول الوزن
- المعاينة تعرض حساب كل نوع شحن مفعّل بشكل منفصل
- إخراج hidden inputs لـ `has_pre_order` و `has_in_stock` و `shipping_type` بصيغة تدعم الاختيارين

```text
┌──────────────────────────────────────────────────┐
│ التسعير بالدولار                                 │
│                                                  │
│ سعر التكلفة ($) | السعر الأصلي قبل التخفيض ($)  │
│                                                  │
│ نوع البيع: ☑ حجز مسبق  ☑ بيع مباشر             │
│                                                  │
│ إعدادات الشحن (حجز مسبق):                       │
│   ☑ بحري → CBM (ط×ع×ا) + عمولة بحري             │
│   ☑ جوي  → الوزن kg + عمولة جوي                 │
│                                                  │
│ البيع المباشر: تكاليف أخرى + عمولة              │
│                                                  │
│ المعاينة: سعر بحري | سعر جوي | سعر مباشر        │
└──────────────────────────────────────────────────┘
```

### 2. تعديل `Admin.tsx` - handleProductSubmit

- قراءة `has_pre_order` و `has_in_stock` من hidden inputs الجديدة (من PricingSection)
- `shipping_type` يمكن أن يكون `'both'` أو `'sea'` أو `'air'`
- حساب السعر النهائي: إذا كان الشحنان مفعلين، يُحسب سعر لكل نوع ويُخزن الأقل أو يُترك للعرض
- حذف حقول السعر والتكلفة والسعر الأصلي من مكانها القديم (السطور 1966-2016) لأنها انتقلت لقسم التسعير

### 3. تعديل `Admin.tsx` - خيارات المنتج والألوان

عند تفعيل البيع المباشر (`has_in_stock`):
- إضافة حقل **سعر البيع المباشر** لكل خيار/لون (حقل منفصل عن `price_adjustment`)
- إضافة حقل **المخزون المتبقي** لكل خيار/لون (موجود حالياً كـ `stock_quantity` ولكن يُعرض دائماً عند تفعيل البيع المباشر)

### 4. حذف حقول السعر والتكلفة القديمة من الفورم الرئيسي

- حذف حقول `price`, `original_price`, `cost_price`, `currency` من مكانها الحالي (السطور 1966-2016)
- نقلها داخل `AdminProductPricingSection` كحقول بالدولار

### 5. الملفات المتأثرة

| ملف | التعديل |
|-----|---------|
| `AdminProductPricingSection.tsx` | إعادة كتابة: checkboxes + نقل حقول السعر |
| `Admin.tsx` | حذف حقول السعر القديمة + تعديل handleProductSubmit |

لا توجد تغييرات على قاعدة البيانات - الأعمدة الحالية كافية.
