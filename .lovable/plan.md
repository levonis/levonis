## الهدف
إصلاح نافذة **تحديث التكلفة السريع** بحيث:
- لا تعرض `0` لتكلفة المنتج الأساسية عندما لا توجد تكلفة فعلية.
- تعمل على تحديث تكلفة المنتج فعلياً.
- تستخدم نفس الحقول الموجودة حالياً فقط، بدون إنشاء حقول جديدة أو ربط جديد.
- تجلب وتعرض تكلفة المنتج، الخيارات، والألوان في النافذة المنبثقة الصغيرة.

## سبب المشكلة الحالي
- النافذة الحالية أصبحت تقرأ `original_price_usd` وتحاول حفظه أيضاً، وهذا يخالف المطلوب الآن لأن المطلوب هو استخدام نفس حقول التكلفة الفعلية الموجودة.
- حفظ المنتج عبر RPC يحدّث `products.cost_price`، لكن الكود أضاف تحديثاً منفصلاً لـ `original_price_usd` وهذا غير مطلوب وقد يسبب عدم اتساق.
- النافذة حالياً تعرض الخيارات فقط من `product_options.price_adjustment` ولا تجلب حقول تكلفة الخيار الأصلية مثل `cost_iqd`، ولا تعرض تكاليف الألوان من `products.colors[].cost_iqd / cost_usd`.

## خطة التنفيذ
1. **إرجاع مصدر تكلفة المنتج الأساسي إلى الحقل الأصلي فقط**
   - قراءة `products_admin.cost_price` فقط للمنتج الأساسي.
   - إذا كانت القيمة `null` أو `0` تعرض الحقل فارغاً وليس `0`.
   - عند الحفظ: الحقل الفارغ يُحفظ `null` وليس `0`.

2. **إزالة أي تحديث غير مطلوب لـ `original_price_usd`**
   - لن يتم تحديث `original_price_usd` من نافذة التحديث السريع.
   - لن يتم إنشاء أي حقول جديدة.
   - النافذة ستتعامل فقط مع حقول التكلفة الحالية.

3. **جلب تكلفة الخيارات من الحقول الأصلية**
   - جلب `product_options.cost_iqd` و `price_adjustment`.
   - العرض يكون من `cost_iqd` إن وجد، وإلا من `price_adjustment` عند الحاجة للحفاظ على التوافق.
   - الحفظ سيحدّث نفس الحقول الحالية: `cost_iqd` و `price_adjustment` حسب منطق RPC الموجود، بدون إضافة حقول.
   - `0` أو الفارغ يظهر كحقل فارغ، والحفظ الفارغ = `null`.

4. **إضافة الألوان داخل نفس النافذة بدون حقول جديدة**
   - جلب `products_admin.colors` من المنتج.
   - عرض كل لون مع تكلفة اللون الموجودة داخل نفس JSON: `cost_iqd` و/أو `cost_usd`.
   - تحديث الألوان يتم بحفظ نفس `colors` JSON بعد تعديل `cost_iqd` فقط للون المحدد.
   - لا يتم إنشاء جدول جديد ولا عمود جديد.

5. **تبسيط إدخال العملة بدون ربط جديد**
   - يمكن اختيار العملة للكتابة: IQD / USD / CNY.
   - عند الحفظ يتم تحويل القيمة إلى دينار وحفظها في نفس حقول التكلفة الحالية.
   - التحويل يستخدم إعدادات الشحن الحالية الموجودة (`usd_to_iqd_rate`, `cny_to_usd_rate`) فقط.
   - لا يتم حفظ العملة نفسها ولا إضافة أعمدة لها.

6. **تحديث RPC إن احتاج فقط ليدعم الألوان داخل `products.colors`**
   - تعديل دالة `admin_quick_update_costs` الموجودة فقط، بدون إنشاء حقول.
   - ستستمر بتحديث `products.cost_price` و `product_options.cost_iqd/price_adjustment`.
   - ستضيف تحديثاً اختيارياً لـ `products.colors` JSON إذا أرسلت النافذة بيانات ألوان.

## النتيجة المتوقعة
- عند فتح النافذة: تكلفة المنتج الأساسية لا تظهر `0` بل فارغة إذا لا توجد قيمة.
- حفظ تكلفة المنتج الأساسي يحدّث الحقل الحقيقي `cost_price`.
- تكلفة الخيار واللون تظهر من نفس حقولها الأصلية وتُحفظ فيها.
- لا يتم إنشاء حقول جديدة ولا تغيير بنية قاعدة البيانات إلا تعديل الدالة الحالية إذا احتجنا تمرير الألوان.
