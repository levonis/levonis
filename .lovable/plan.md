

## ملخص التغيير

تحسين جذري لصفحة تفاصيل المنتج مع إضافة نظام اختيار نوع البيع (مباشر/حجز مسبق)، ربط الألوان بالخيارات، وإصلاح مشكلة خصومات البطاقات.

---

## التفاصيل التقنية

### 1. إصلاح مشكلة خصومات البطاقات (Admin.tsx)
- السطر 2041: تغيير `step="500"` إلى `step="1"` لأن القيمة الحالية لا تسمح بإدخال أرقام مثل 250
- أيضاً `value={discount.discount_amount}` عندما تكون القيمة 0 لا تسمح بالكتابة - يجب تغييرها إلى `value={discount.discount_amount || ''}`

### 2. ربط الألوان بالخيارات في الأدمن (Admin.tsx)
إضافة حقل جديد لكل لون في قسم الألوان:
- **"الخيارات المتاحة لهذا اللون"**: مجموعة checkboxes بأسماء الخيارات الموجودة
- يتم تخزين مصفوفة `linked_options` كـ array من أسماء الخيارات داخل كائن اللون في JSON
- إضافة حقل `direct_sale_price` و `stock_quantity` لكل لون عند تفعيل البيع المباشر

تعديل `productColors` state type:
```text
linked_options?: string[]  // أسماء الخيارات المرتبطة (فارغ = متاح لكل الخيارات)
direct_sale_price?: number
```

### 3. إعادة تصميم صفحة تفاصيل المنتج (ProductDetail.tsx)
التصميم الجديد:

```text
┌─────────────────────────────────────────────────┐
│ صور المنتج (carousel محسّن)                     │
├─────────────────────────────────────────────────┤
│ اسم المنتج + الوصف                              │
│                                                  │
│ ┌─── اختيار نوع البيع (tabs) ──┐                │
│ │ 🛒 بيع مباشر  │ 📦 حجز مسبق │                │
│ └──────────────────────────────┘                │
│                                                  │
│ [إذا بيع مباشر]:                                 │
│   الخيارات → الألوان (مفلترة حسب الخيار)        │
│   السعر + المخزون                                │
│                                                  │
│ [إذا حجز مسبق]:                                  │
│   نوع الشحن (بحري/جوي)                          │
│   الخيارات → الألوان (مفلترة)                    │
│   السعر النهائي حسب الشحن                        │
│                                                  │
│ الكمية + إضافة للسلة                             │
│ خصومات البطاقات + النقاط                         │
└─────────────────────────────────────────────────┘
```

**التغييرات الرئيسية:**
- إضافة state `selectedSaleType` بدل عرض كلا النوعين معاً
- Tabs أنيقة لاختيار نوع البيع (تظهر فقط إذا كان المنتج يدعم أكثر من نوع)
- فلترة الألوان حسب الخيار المحدد باستخدام `linked_options`
- عرض سعر مختلف حسب نوع البيع (البيع المباشر يستخدم `direct_sale_price` إن وُجد)
- تحسين عام للتنسيق: تصميم أنظف وأكثر تنظيماً

### 4. منطق فلترة الألوان حسب الخيار

```text
إذا اللون لديه linked_options فارغ/غير موجود → متاح لكل الخيارات
إذا linked_options يحتوي اسم الخيار المحدد → اللون متاح
غير ذلك → اللون غير متاح (يظهر خافت)
```

### 5. الملفات المتأثرة

| ملف | التعديل |
|-----|---------|
| `Admin.tsx` | إصلاح step خصومات البطاقات + إضافة linked_options و direct_sale_price للألوان |
| `ProductDetail.tsx` | إعادة تصميم كاملة مع tabs نوع البيع + فلترة الألوان |

لا حاجة لتغييرات في قاعدة البيانات - البيانات تُخزن في عمود `colors` الموجود (JSON).
